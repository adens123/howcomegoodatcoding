Function的時間複雜度、Object語法

作業日期：2023/09/26

Function的時間複雜度

  • 在電腦科學中，演算法的時間複雜度（Time complexity）可以描述該演算法的執行時間。 時間複雜度常用大O符號表述，不包括這個函式的低階項和首項係數。

  • 直觀上來說，時間複雜度就是，在參數逐漸變大的情況下，Function執行完畢所需要的時間的成長速度。

  • JavaScript陣列中的常見method相對應的時間複雜度為：

    • push – O(1)

    • pop – O(1)

    • shift – O(n)

    • unshift – O(n)

  • 每個瀏覽器的 JavaScript 引擎內部對於陣列的實現方法略有差異。對於不同大小的陣列，可能會採用其他更好的資料結構，例如double-linked list, binary search tree (BST)等等，來增強表現。

Object 物件

  • 每個 JavaScript 物件都有properties以及method。屬於物件的function被稱為method。 物件的屬性與相對應的值是一種key-value pair。獲取物件屬性的方式可以透過dot notation或是 []。JavaScript Object是一種hashtable。

    let Panda = {
     name : "Fubo",
     age: 5,
     sayHello(){
      alert("Hi, I'm Fubo!")
     }
    } // 建立一個Panda的物件，sayHello()是這個物件裡面的function，並且是一個method。
    console.log(Panda.age); // 5
    console.log(Panda["name"]);// Fubo

  • 在method中的this關鍵字指的是調用該方法的物件。若某個function沒有調用該function的物件，則this關鍵字則是指向window物件。

  • 在 JavaScript 當中的function、array 其實都是物件(Object)。Array以及Function都是special type of objects。
